# Prospector Universal — Configuration Guide

## First Run

### 1. Copy the configuration file

```bash
cp config.json.example config.json
```

### 2. Fill in your data

Edit `config.json` with your information:

```json
{
  "assinatura": {
    "nome": "John Doe",
    "apresentacao": "Digital Presence Specialist for Healthcare Professionals",
    "whatsapp": "5511999999999"
  },
  "prospeccao": {
    "nichos": ["nutritionists", "psychologists"],
    "cidade": "São Paulo",
    "leadsPorBusca": 10
  }
}
```

### 3. Configure hosting

Choose your provider from `hosting/` and follow the specific instructions. Example for Netlify:

```bash
# Install Netlify CLI (optional, for manual deploy)
npm install -g netlify-cli
```

Then set the provider in `config.json`:

```json
{
  "hospedagem": {
    "provedor": "netlify",
    "pastaBase": "clients"
  }
}
```

### 4. (Optional) Connect your Gmail

To send proposals via email, you can:
- Use manual draft (copy the text generated by AI)
- Use Gmail integration (if your AI assistant supports it)
- Export data to your preferred email tool

### 5. Ready!

Now open `prompts/1-setup.md` and share with your preferred AI assistant.

---

## Managing Credentials

⚠️ **Security**: Your hosting password is stored in plain text in `config.json` on your computer.
Never share this file. Never paste the password directly in the AI chat.

### Best practices:
- Keep `config.json` in `.gitignore`
- Don't version this file
- Use environment variables if possible

---

## Migrating from the Original Prospector de Sites

If you already use the original Claude version:

1. Copy your data from `prospector-config.json` to the new `config.json`
2. Adjust the hosting provider (no longer required to be HostGator)
3. Leads in `leads.md` are compatible — keep the file
4. Folders in `sites/[slug]/` are compatible — keep them
